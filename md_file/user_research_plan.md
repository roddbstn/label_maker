# 사용자 리서치 및 테스트 시나리오 설계

본 문서는 라벨 메이커 서비스의 핵심 가치(평균 업무 시간 50% 단축, 사용성 검증, 추천 의향)를 확인하기 위한 인터뷰 및 테스트 시나리오를 담고 있습니다.

## 1. 리서치 목적
- **효율성 검증**: 기존 방식(HWP 등) 대비 업무 시간이 실제로 50% 이상 단축되는가?
- **사용성 확인**: 핵심 기능(자동 피팅, PDF 생성 등) 이용 시 허들이나 오류는 없는가?
- **바이럴 가능성**: 사용자가 서비스에 만족하며 주변 동료에게 추천할 의사가 있는가?

---

## 2. 사전 인터뷰 (기존 방식 파악)
사용자가 기존에 라벨을 어떻게 만들었는지 파악하여 '라벨 메이커'와의 차이를 극명하게 대조합니다.

1. **현재 업무 현황**: "평소 정부문서화일(황대일 파일) 라벨 작업을 얼마나 자주, 한 번에 몇 장 정도 하시나요?"
2. **기존 방식**: "라벨 작업을 할 때 어떤 도구(한글 양식, 폼텍 프로그램 등)를 사용하시나요?"
3. **페인 포인트**: "기존 방식에서 가장 번거롭거나 스트레스 받는 부분은 무엇인가요? (예: 장평 조절, 폰트 깨짐 등)"
4. **소요 시간**: "라벨 10장을 뽑는다고 가정했을 때, 오타 검수부터 출력까지 총 몇 분 정도 소요되시나요?"

---

## 3. 유저 테스트 (UT) 시나리오
사용자가 서비스의 핵심 가치를 직접 경험하게 합니다. (Think-aloud: 생각나는 것을 소리 내어 말하며 진행 유도)

### [Task 1] 단일 라벨 생성 및 자동 피팅 경험
- **상황**: "새로운 서류철을 만들려고 합니다. 아래 정보를 입력하고 미리보기가 어떻게 나오는지 확인해 보세요."
- **입력 정보**: 제목이 아주 긴 부서명(예: 대전광역시 아동보호전문기관 특화사업지원팀)
- **확인 포인트**: 사용자가 글자가 넘칠 때 당황하는지, 아니면 자동으로 줄어드는 것을 보고 안도하는지 관찰.

### [Task 2] 다중 라벨 편집 및 PDF 저장
- **상황**: "추가로 2개의 라벨을 더 만들어야 합니다. '라벨 추가' 기능을 사용해 다른 제목을 입력하고, 최종적으로 PDF 파일을 다운로드해 보세요."
- **확인 포인트**: 탭 전환의 편리함, '+' 버튼의 발견 용이성, PDF 생성 속도에 대한 반응.

### [Task 3] 가이드 및 초기화 활용
- **상황**: "기능이 헷갈리는 상황입니다. 가이드를 확인하고, 입력한 내용을 모두 지운 뒤 새로 시작해 보세요."
- **확인 포인트**: 가이드 오버레이의 명확성, 초기화 버튼의 위치 및 동작 기대치 부합 여부.

---

## 4. 사후 인터뷰 (가치 검증 및 추천)
테스트 종료 후 서비스의 비즈니스 임팩트를 측정합니다.

1. **시간 단축 체감**: "기존 방식과 비교했을 때, 10장 출력 기준 작업 시간이 얼마나 줄어들 것 같나요? 50% 이상 단축이 가능해 보이시나요?"
2. **가장 좋았던 기능**: "오늘 사용해 본 기능 중 가장 인상 깊었거나 꼭 필요하다고 느낀 기능은 무엇인가요?"
3. **불편 및 개선점**: "방금 작업을 하면서 '이건 좀 이상하다'거나 '이게 있으면 더 좋겠다'고 생각하신 부분이 있나요?"
4. **추천 의향 (NPS)**: "이 서비스를 주변 사회복지사/공무원 동료에게 추천하시겠습니까? 0점(절대 안 함)부터 10점(반드시 함) 사이로 점수를 매긴다면 몇 점인가요? 그 이유는?"

---

## 5. 관찰 체크리스트 (기획자용)
- [ ] 제목 입력 시 장평이 조절되는 시점에 사용자가 어떤 반응(미소, 감탄, 질문 등)을 보이는가?
- [ ] '양식 미리보기'와 '화일 미리보기' 탭을 번갈아 보며 내용을 확인하는가?
- [ ] 버튼 아이콘과 텍스트가 행동을 직관적으로 유도하는가? (예: 🖨️ 바로 인쇄)
- [ ] PDF를 열었을 때 실제 출력물과 같을 것이라는 신뢰를 느끼는가?
