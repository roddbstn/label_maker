## 2. 문제 강도 (Problem Intensity)

### 문제가 없으면 타겟에게 어떤 일이 벌어지나?
- **업무 중단 X**: 라벨을 안 붙여도 업무는 돌아감
- **다만**: 귀찮고 시간이 낭비됨 (매번 10~20분 소요)
- **감사 대비**: 문서 관리 미흡 지적 가능성 (간접적 리스크)

### 현재 타겟은 이 문제를 어떻게 해결하고 있나?
1. 인터넷에서 HWP 양식 다운로드 → 직접 수정
2. 부서 내 기존 파일을 복사해서 수정
3. 손글씨로 작성 (일부)
4. 폼텍 디자인 프로 9 설치 후 사용 (소수)

### 기존 해결책에 돈/시간을 얼마나 쓰고 있나?
| 항목 | 비용 |
|------|------|
| 금전적 비용 | **₩0** (무료 양식, 무료 프로그램) |
| 시간적 비용 | **회당 10~30분** × 주 1~2회 = **월 1~4시간** |
| 정서적 비용 | 짜증, 스트레스 (반복 노동에 대한 피로) |

### 문제 강도 점수

## ⭐ 4/10

**이유:**
- ❌ **생사가 걸린 문제가 아님** (Nice-to-have)
- ❌ **현재 무료로 해결 중** (돈을 쓸 동기 약함)
- ❌ **발생 빈도 낮음** (주 1~2회)
- ⚠️ **개인이 아닌 조직의 문제** (의사결정 구조 복잡)
- ✅ 다만, 불편함은 확실히 존재
- ✅ 대량 출력이 필요한 시기(연말/연초)에는 강도 상승

## 3. 솔루션 적합성 (Solution Fit)

### 이 솔루션이 문제를 해결하는 방식이 논리적인가?
✅ **논리적임**
- 입력 → 자동 피팅 → PDF 출력의 흐름은 명확
- 자동 장평/자간 조절은 핵심 페인포인트 해결

### 타겟이 이 방식을 받아들일 준비가 되어 있나?
⚠️ **조건부 준비됨**
- 웹 기반 = 설치 불필요 → 공공기관 보안 환경에 적합 ✅
- 그러나: 새 서비스 학습에 대한 저항 가능 (기존 습관)
- 그러나: 조직 내 IT 정책으로 외부 서비스 접근 제한 가능

### 기술적으로 구현 가능한가?
✅ **충분히 가능**
- Canvas API 또는 PDF 라이브러리로 구현 가능
- 텍스트 피팅 알고리즘 복잡도 중간
- 엑셀 파싱은 라이브러리 활용

### 솔루션 적합성 점수

## ⭐ 7/10

**이유:**
- ✅ 기술적 난이도 낮음
- ✅ 문제-솔루션 연결고리 명확
- ✅ 웹 기반 = 진입장벽 낮음
- ⚠️ 유료 전환 명분이 약함 (무료로도 되는 일)
- ⚠️ 타겟의 IT 리터러시 다양함

## 4. 타이밍 (Why Now?)

### 왜 지금 이 서비스가 필요한가?
- ❓ **특별히 "지금이어야 할" 이유가 약함**
- 이 문제는 10년 전에도 존재했고, 10년 후에도 존재할 것
- 종이 문서 → 전자문서 전환 추세로 오히려 시장 축소 가능

### 이전에 비슷한 시도가 실패했다면 그 이유는?
- 직접적인 실패 사례 확인 불가
- 다만, 라벨 회사들(폼텍, 프린텍)이 비슷한 프로그램을 만들고도 **업데이트를 멈춤** → 수익성 부족 시사

### 지금 가능하게 만드는 기술/환경 변화는?
| 변화 | 영향 |
|------|------|
| 웹 기술 발전 | PDF 생성, Canvas API 등 브라우저에서 가능 |
| AI 코딩 툴 | 1인 개발로 MVP 가능 (개발비 절감) |
| 클라우드 인프라 | 저렴한 서버 운영 (Vercel, Supabase 등) |

> [!CAUTION]
> **타이밍 리스크:** 정부의 전자문서 전환 정책(문서24 등)이 가속화되면 **종이 라벨 자체의 수요가 감소**할 수 있음. 3~5년 후 시장 축소 가능성 존재.

## 6. 최종 판정

# 🟡 Pivot (조건부 진행)

### 판정 이유 (3가지)

1. **문제 강도 부족 (4/10)**: "있으면 좋고, 없어도 사는 데 지장 없는" 문제. 무료 대안이 이미 존재하며, 유료 전환 동기가 약함.

2. **수익화 모델 불확실**: 타겟이 개인이 아닌 조직이지만, 조직이 돈을 쓸 만큼의 문제가 아님. 개인 결제도 기대하기 어려움 (월 ₩5,900을 라벨 출력에?).

3. **시장 타이밍 불리**: 종이 문서 → 전자문서 전환 추세. 장기적으로 시장 축소 가능성.

---

### Pivot 방향 제안

> [!TIP]
> **핵심 방향:** "수익화 목표"를 낮추고, **포트폴리오/사이드 프로젝트**로 접근하거나, **더 큰 문제**로 확장

#### Option A: 사이드 프로젝트로 진행
- 수익 기대 X, 완전 무료
- 개발 경험 + 포트폴리오 + 사용자 반응 테스트용
- 2주 내 초간단 MVP만 출시

#### Option B: 문제 확장
- 라벨 출력 → **문서 관리 자동화** 전체로 확장
- 서류철 표지 생성 + 목차 생성 + 라벨 출력 통합
- 조직 단위 SaaS로 전환 (B2B)

#### Option C: 타겟 변경
- 정부문서화일 → **소상공인 제품 라벨** (식품, 화장품 등)
- 이 시장이 더 크고, 유료 결제 의향 높음
